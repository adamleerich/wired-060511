WIRED - Windows Registry Dataset

Executive Summary

WIRED is both a Windows Registry Dataset and a set of tools for creating the dataset. The dataset contains the changes to the Windows Registry caused by application installation, de-installation, execution or other Registry modifying operations. The applications are chosen to be of interest to computer forensic examiners.

Current Status

WIRED is currently just an experimental prototype. We are planning to solicit feedback from the computer forensics community to improve and extend its usefulness.

The Dataset

The dataset can be found here.

The WIRED dataset currently has the following fields:

  • CHANGE_TYPE - whether the Registry entry was added, deleted or modified
  • APP_NAME - the application's name
  • NSRL_APP_ID - if the application is part of the NSRL, its NSRL app ID
  • ACTION - whether the application was installed, deinstalled, executed or some other type of registry modification occurred
  • ENTRY_TYPE - is the Registry entry a key or value?
  • PATH - the Registry entry's path
  • VALUE_NAME - if the entry is a value, its name
  • VALUE_DATA - if the entry is a value, the data it contains

The Tools

There are currently two tools:

  1. reg-diff.rb - generates a XML-based difference between two Microsoft RegEdit-generated Registry patch files.
  2. mk-dataset.rb - creates the WIRED dataset from difference files generated by reg-diff.rb.

They can be found here. The WIRED tools are currently implemented in Ruby (1.8.4) and were tested in Mac OS X 10.4 (Tiger). They should work with other BSD-style operating systems, but we haven't tried that yet.

Documentation for both of these tools and associated libraries are here.

Caveats and Future Directions

WIRED is currently just an experimental prototype. We've run into the limitations of using RegEdit to generate Registry dumps and will have to better handle the problematic Registry entries. Currently, reg-diff.rb will issue warnings like the following for text-based entries with embedded binary data that don't translate well to RegEdit's text-based output:

Warning: Ignoring badly formatted value found in SetupRasputin.reg: [HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\W32Time] "=

We've also only begun identifying, acquiring and processing software for inclusion in the dataset. Much of it is not the type of software currently found in the NSRL, but exists as shareware and freeware downloadable from the Web.

Finally, we envision the current prototype as only a small step in a much larger scheme that includes:

  • An XML database for managing the Registry difference files. This will allow for the efficient query and manipulation of acquired Registry data.
  • Direct interfaces to the Registry data via Microsoft's APIs.
  • Acquistion and cryptographic hashing of all files installed or modified by an application of interest.
  • Expansion of Registry modification detection to beyond just application installation to include all phases of an application's life cycle on a given machine.
  • Investigation of installation and testing automation to improve our ability to keep up with the explosion of new software found on the Web.
